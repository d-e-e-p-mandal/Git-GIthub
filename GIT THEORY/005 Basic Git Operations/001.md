# 5. Basic Git Operations

# Checking Repository Status

# What is Repository Status?

Repository status shows the current condition of files inside the Git repository.

Git status helps developers know:
- Which files changed
- Which files are staged
- Which files are untracked
- Current branch information

---

# Purpose of Checking Status

Used to:
- Monitor changes
- Prepare commits
- Avoid mistakes
- Track workflow progress

---

# Status Command

```bash
git status
```

---

# Short Status Command

```bash
git status -s
```

---

# Example Output

```text
On branch main
Changes not staged for commit:
modified: app.js

Untracked files:
style.css
```

---

# File States in Status

| State | Meaning |
|---|---|
| Untracked | New file |
| Modified | Changed file |
| Staged | Ready for commit |
| Committed | Saved permanently |

---

# Importance of Status Command

The `git status` command is one of the most frequently used Git commands because it helps developers understand repository condition before performing operations.

---

# Adding Files

Adding files means telling Git to start tracking files or prepare modified files for commit.

Files move from:
- Working Directory
to
- Staging Area

---

# Purpose of Adding Files

Used to:
- Track new files
- Stage modifications
- Prepare commits

---

# Add Single File

```bash

```

---

# Add Multiple Files

```bash

```

---

# Add All Files

```bash

```

---

# Add All Changes

```bash

```

---

# Add Modified & Deleted Files

```bash

```

---

# Add Using Wildcards


---

# What Happens After Adding?

Git:
- Tracks file
- Stores staged snapshot
- Prepares commit data

# Benefits of Adding Files

## 1. Selective Commits

Choose exact files.

---

## 2. Organized History

Separate unrelated changes.

---

## 3. Better Collaboration

Cleaner commit structure.

Staging means preparing changes before committing.

The staging area acts like:
- Temporary preparation zone

---

# Purpose of Staging

Allows developers to:
- Review changes

- Control commit contents

# Committing Changes

# What is Commit?

A commit is a permanent snapshot of staged changes.

Commits save:
- File states
- Metadata
- Project history

---

# Purpose of Commits

Commits are used to:
- Save progress
- Track changes
- Restore versions
- Collaborate with teams

---

# Basic Commit Command

```bash
git commit
```

---

# Commit with Message

```bash
git commit -m "Add login feature"
```

---

# Commit All Tracked Changes

```bash
git commit -a -m "Update project"
```

---

# Amend Last Commit

```bash
git commit --amend
```

---

# Commit Structure

Each commit contains:
- Commit hash
- Author
- Timestamp
- Commit message
- Snapshot reference

---

# Commit Workflow

```text
Edit Files
        ↓
Stage Changes
        ↓
Commit Changes
```

---

# Importance of Frequent Commits

Frequent commits:
- Improve recovery
- Simplify debugging
- Maintain clean history

---

# Undoing Changes

# What is Undoing Changes?

Undoing changes means reverting unwanted modifications.

Git allows undoing:
- Working changes
- Staged changes
- Commits

---

# Undo Working Directory Changes

```bash
git restore file-name
```

---

# Undo Staged Changes

```bash
git restore --staged file-name
```

---

# Undo Last Commit (Keep Changes)

```bash
git reset --soft HEAD~1
```

---

# Undo Last Commit (Remove Changes)

```bash
git reset --hard HEAD~1
```

---

# Revert Specific Commit

```bash
git revert commit-id
```

---

# Restore Deleted File

```bash
git restore file-name
```

---

# Important Warning

Hard reset permanently removes changes unless recoverable through reflog.

---

# Removing Files

# What is Removing Files?

Removing files deletes files from:
- Working directory
- Git tracking system

---

# Remove File

```bash
git rm file-name
```

---

# Remove Cached File Only

```bash
git rm --cached file-name
```

Keeps file locally but removes tracking.

---

# Remove Folder

```bash
git rm -r folder-name
```

---

# Force Remove

```bash
git rm -f file-name
```

---

# Remove Ignored Tracking

Useful for:
- `.env`
- build files
- temporary files

---

# Benefits of git rm

- Removes tracking properly
- Updates Git history
- Synchronizes deletion

---

# Renaming Files

# What is Renaming Files?

Renaming changes file name while preserving Git history.

---

# Rename File

```bash
git mv old-name new-name
```

---

# Example

```bash
git mv app.js main.js
```

---

# What Happens Internally?

Git performs:
1. Rename
2. Stage rename automatically

---

# Benefits of git mv

- Preserves history
- Cleaner workflow
- Automatic staging

---

# Renaming Folder

```bash
git mv old-folder new-folder
```

---

# Moving Files

# What is Moving Files?

Moving files changes file location inside repository while preserving history.

---

# Move File

```bash
git mv file-name folder/
```

---

# Example

```bash
git mv app.js src/
```

---

# Move Multiple Files

```bash
git mv *.js src/
```

---

# Move Folder

```bash
git mv old-folder new-folder
```

---

# Benefits of git mv

| Benefit | Description |
|---|---|
| History Preservation | Tracks movement |
| Automatic Staging | No extra add required |
| Cleaner Workflow | Simplified management |
