# 8. Merging in Git

# Merge Concept

# What is Merging?

Merging in Git means combining changes from one branch into another branch.

It allows developers to:
- Combine features
- Integrate bug fixes
- Synchronize development work
- Update production code

---

# Purpose of Merging

Merging is used to:
- Bring feature code into main branch
- Combine team contributions
- Integrate parallel development
- Maintain project continuity

---

# Basic Merge Example

Suppose:

```text
main
 ├── Commit A
 ├── Commit B
 │
 └── feature-login
       ├── Commit C
       └── Commit D
```

After merge:

```text
main
 ├── Commit A
 ├── Commit B
 ├── Commit C
 └── Commit D
```

---

# Merge Workflow

Typical workflow:

```text
Create Branch
        ↓
Develop Feature
        ↓
Commit Changes
        ↓
Switch to Main
        ↓
Merge Branch
```

---

# Merge Command

```bash
git merge branch-name
```

Example:

```bash
git merge feature-login
```

---

# Important Merge Notes

Merging:
- Preserves history
- Combines commit changes
- Updates target branch

---

# Types of Merge

Git mainly supports:
1. Fast Forward Merge
2. Three-Way Merge

---

# Fast Forward Merge

# What is Fast Forward Merge?

Fast forward merge happens when:
- Target branch has no new commits
- Git simply moves branch pointer forward

---

# Fast Forward Merge Example

Before merge:

```text
main
 ├── A
 └── B
       \
        feature
          ├── C
          └── D
```

After merge:

```text
main
 ├── A
 ├── B
 ├── C
 └── D
```

No extra merge commit created.

---

# Characteristics

| Feature | Fast Forward Merge |
|---|---|
| Merge Commit | No |
| History | Linear |
| Complexity | Simple |
| Speed | Faster |

---

# How Git Performs Fast Forward

Git:
- Moves main branch pointer
- No branch divergence exists

---

# Fast Forward Merge Command

```bash
git merge feature-branch
```

If conditions allow:
- Git performs fast forward automatically

---

# Prevent Fast Forward Merge

```bash
git merge --no-ff branch-name
```

Creates merge commit even if fast forward possible.

---

# Benefits of Fast Forward Merge

## 1. Cleaner History

Linear commit history.

---

## 2. Simpler Graph

Easy to understand.

---

## 3. Faster Merge

No extra commit needed.

---

# Limitations

History may hide:
- Feature branch existence
- Branch workflow details

---

# Three-Way Merge

# What is Three-Way Merge?

Three-way merge occurs when:
- Both branches contain new commits
- Branch histories diverged

Git combines:
1. Common ancestor
2. Source branch
3. Target branch

---

# Three-Way Merge Example

Before merge:

```text
        ├── C (main)
A ── B
        └── D (feature)
```

After merge:

```text
        ├── C
A ── B ───────── M
        └── D
```

`M` = Merge Commit

---

# Why Called Three-Way?

Git compares:
1. Common ancestor
2. Current branch
3. Merging branch

---

# Characteristics

| Feature | Three-Way Merge |
|---|---|
| Merge Commit | Yes |
| History | Non-linear |
| Preserves Branch History | Yes |

---

# Three-Way Merge Process

Git:
1. Finds common ancestor
2. Compares changes
3. Combines modifications
4. Creates merge commit

---

# Benefits

- Preserves complete history
- Shows branch structure
- Better collaboration visibility

---

# Merge Commit

# What is Merge Commit?

A merge commit is a special commit created during:
- Three-way merge
- Non-fast-forward merge

It combines histories of multiple branches.

---

# Characteristics of Merge Commit

| Property | Description |
|---|---|
| Multiple Parents | Usually 2 parent commits |
| Special Commit | Represents merge point |
| Preserves History | Maintains branch records |

---

# Merge Commit Structure

```text
Commit M
 ├── Parent 1 → main
 └── Parent 2 → feature
```

---

# Example Merge Commit Message

```text
Merge branch 'feature-login'
```

---

# Create Merge Commit Forcefully

```bash
git merge --no-ff branch-name
```

---

# Benefits of Merge Commits

## 1. Preserves Feature History

Shows exact integration point.

---

## 2. Better Collaboration Tracking

Team changes easier to analyze.

---

## 3. Improves Project Auditability

Clear development structure.

---

# Merge Strategies

# What are Merge Strategies?

Merge strategies define how Git combines histories and file changes.

---

# Common Merge Strategies

| Strategy | Purpose |
|---|---|
| Recursive | Default strategy |
| Resolve | Simple merges |
| Octopus | Multiple branch merges |
| Ours | Keep current branch changes |
| Subtree | Merge repositories |

---

# Recursive Strategy

Default strategy for:
- Two branch merges

Handles:
- Complex histories
- Renames
- Multiple ancestors

---

# Resolve Strategy

Used for:
- Simple two-branch merges

Less advanced than recursive.

---

# Octopus Strategy

Used for:
- Merging multiple branches simultaneously

Example:

```bash
git merge branch1 branch2 branch3
```

---

# Ours Strategy

Keeps:
- Current branch changes only

Ignores:
- Incoming branch content

---

# Subtree Strategy

Used when:
- One repository inside another repository

---

# Merge Strategy Command

```bash
git merge -s strategy-name branch-name
```

Example:

```bash
git merge -s ours feature-login
```

Regularly pull latest changes.

# 3. Commit Frequently

# 4. Write Clear Commit Messages

Improves merge readability.

---

# 5. Test Before Merge

Ensure branch stability.

Encourages code review.

# 7. Delete Merged Branches

Keep repository clean.

- Conflicts

- Same lines modified differently

- Resolve conflicts manually

# Merge vs Rebase

| Merge | Rebase |
|---|---|
| Preserves history | Rewrites history |
| Creates merge commits | Linear history |
| Safer collaboration | Cleaner history |
